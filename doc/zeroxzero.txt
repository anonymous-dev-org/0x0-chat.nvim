1. INTRODUCTION                                       *zeroxzero-intro*

0x0.nvim is a Neovim companion for the 0x0 AI coding assistant. The TUI
is the brain. Neovim is the hands. This plugin bridges the two — send
file context to the TUI, review diffs natively in vimdiff, and make
inline code edits without leaving your editor.

The plugin connects to the 0x0-server daemon via HTTP and SSE for
permission dialogs, question dialogs, file reload events, toast
notifications, and statusline updates.

==============================================================================
2. SETUP                                              *zeroxzero-setup*

Requirements:
- Neovim >= 0.10
- `curl` in PATH
- `0x0-server` installed (https://github.com/anonymous-dev-org/0x0)
- `ANTHROPIC_API_KEY` env var set (or configured in config.yaml)

Using lazy.nvim: >lua
  {
    "anonymous-dev-org/0x0.nvim",
    opts = {},
  }
<

Using packer.nvim: >lua
  use {
    "anonymous-dev-org/0x0.nvim",
    config = function()
      require("zeroxzero").setup()
    end,
  }
<

Manual setup: >lua
  require("zeroxzero").setup({
    -- see |zeroxzero-config| for options
  })
<

==============================================================================
3. CONFIGURATION                                      *zeroxzero-config*

Default configuration: >lua
  require("zeroxzero").setup({
    cmd = "0x0-server",             -- Server binary (falls back to "0x0 serve")
    port = 4096,                    -- Server port
    hostname = "127.0.0.1",         -- Server hostname
    auto_start = true,              -- Start server if not running
    keymaps = {
      send = "<leader>0s",          -- Send file (n) or selection (v) to TUI
      send_message = "<leader>0S",  -- Send context + message to TUI
      diff = "<leader>0d",          -- Review diffs in vimdiff
      interrupt = "<leader>0i",     -- Interrupt current response
      inline_edit = "<leader>0e",   -- Inline edit
    },
    auth = nil,                     -- { username = "...", password = "..." }
  })
<

To disable a keymap, set it to empty string in setup.

==============================================================================
4. COMMANDS                                           *zeroxzero-commands*

  :ZeroSend                                           *:ZeroSend*
    Send current file reference to the TUI prompt.
    In visual mode, sends the selection with line references.

  :ZeroSendMessage                                    *:ZeroSendMessage*
    Send file/selection context plus a typed message to the TUI
    prompt. Opens vim.ui.input for the message.

  :ZeroDiff                                           *:ZeroDiff*
    Review file diffs from the latest completed session in native
    vimdiff splits. If multiple files were changed, opens a picker.
    If multiple sessions exist, opens a session picker first.

  :ZeroInterrupt                                      *:ZeroInterrupt*
    Interrupt the current response.

  :ZeroInlineEdit                                     *:ZeroInlineEdit*
    Inline edit at cursor position or visual selection.
    Creates a temporary session, sends the edit instruction,
    and auto-reloads the file when done.

==============================================================================
5. KEYMAPS                                            *zeroxzero-keymaps*

Global keymaps (configurable via setup):

  <leader>0s     Normal      Send file reference to TUI
  <leader>0s     Visual      Send selection to TUI
  <leader>0S     Normal/Vis  Send context + message to TUI
  <leader>0d     Normal      Review diffs in vimdiff
  <leader>0i     Normal      Interrupt current response
  <leader>0e     Normal/Vis  Inline edit

==============================================================================
6. STATUSLINE                                         *zeroxzero-statusline*

Add 0x0 status to your statusline: >lua
  -- lualine example
  sections = {
    lualine_x = {
      { function() return require("zeroxzero").statusline() end },
    },
  }
<

Returns a string with connection status, animated spinner when busy,
and current session title. Returns empty string when not connected.
